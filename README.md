## NEWSPAPER 프로젝트

이 저장소는 로컬 신문사를 위한 기사 수집·편집·배포용 웹 애플리케이션입니다.  
Spring Boot 기반 API 서버와 React(Vite) 프론트엔드로 구성되어 있으며, MinIO(S3 호환 저장소)를 이용해 이미지/배너 자산을 관리합니다.

---

## 전체 구조

- **백엔드**: `src/api-server`
  - Spring Boot 기반 REST API
  - MySQL(또는 호환 DB)을 사용한 기사/이미지 데이터 영속화
  - MinIO(S3 API)로 이미지 업로드/삭제 처리
- **프론트엔드**: `src/newspaper-frontend`
  - React + Vite + TailwindCSS 스타일
  - 기사 리스트/헤드라인/추천 슬라이더/사이드배너/관리자 페이지 UI
- **데이터 디렉터리**: `data/`
  - 로컬 개발용 MySQL, MinIO 데이터가 저장되는 경로 (Git에 커밋하지 않는 것이 권장됨)

## 주요 기능 정리

### 1. 기사 목록 및 상세 보기

- **카테고리별 뉴스 목록**
  - 상단 헤더의 카테고리(정치/경제/사회/문화/교육/인터뷰칼럼/경기도소식/동영상)를 기준으로 기사 필터링
  - 카테고리 변경 시 작성 중인 내용이 있으면 경고 후 이동 여부를 선택
  - 하단 페이지네이션:
    - 최대 10개의 페이지 번호 버튼
    - 이전/다음 버튼
    - 페이지 번호를 직접 입력 후 `이동`으로 점프
- **헤드라인 섹션**
  - 최신 기사 1건을 헤드라인으로 표시
  - 좌측: 업로드된 본문 이미지 기반 썸네일
  - 우측: 카테고리 배지, 제목, 긴 요약 텍스트(HTML 본문에서 텍스트 추출)
- **섹터별 추천(그룹 위젯)**
  - 정치/경제, 사회/문화, 교육, 인터뷰/경기도소식 등 섹터별로 최대 3개의 추천 기사 노출
  - 각 카드: 좌측 썸네일 + 우측 제목/요약/카테고리/날짜/기자

### 2. 기사 작성/수정/삭제 (관리자)

- **관리자 로그인**
  - 프론트: `LoginModal`에서 아이디/비밀번호 입력
  - 백엔드: `/api/admin/login`으로 검증
    - 하드코딩 계정: `admin / 8593`
  - 로그인 성공 시 로컬스토리지에 `isAdmin=true` 저장, 관리 기능 활성화
- **기사 작성**
  - 카테고리 선택 + 제목 입력 + Quill 기반 본문 편집기
  - 본문 내 이미지는 S3(MinIO)에 미리 업로드된 URL을 사용하는 구조(에디터에 직접 `<img>` 삽입)
  - 작성 중에는 전역 플래그(`window.__articleDirty`)로 변경 상태를 관리하여, 카테고리 이동 시 경고
  - 저장 시:
    - 본문에서 `<img src="...">`를 파싱해 `imageUrls`로 백엔드에 전달
    - DB에는 기사 + 이미지 엔티티로 저장
    - 프론트 기사 리스트 상태를 즉시 업데이트
- **기사 수정**
  - 목록 화면에서 각 기사 우측의 `수정` 버튼 클릭 시 동일 폼으로 전환
  - 상세 화면에서도 상단 헤더(제목/공유 옆)의 `수정` 버튼으로 직접 편집 가능
  - 수정 시 기존 이미지 리스트를 교체하는 방식으로 `imageUrls` 업데이트
- **기사 삭제**
  - 목록 화면, 상세 화면 모두에서 삭제 가능
  - 백엔드 `/api/articles/{id}` DELETE 호출 후 프론트 상태/화면에서 제거

### 3. 추천 이미지/썸네일 처리

- **기본 정책**: 랜덤 더미 이미지 사용 없이, **반드시 업로드된 이미지**만 사용
- 기사 응답의 `content` HTML에서 첫 번째 `<img src="...">`를 정규식으로 추출하여 썸네일로 사용
  - 헤드라인, 카테고리 리스트, 섹터별 추천 카드, 오른쪽 추천 슬라이더, 상세 페이지 하단 추천 모두 동일 기준 사용
- 이미지 크기/비율
  - 배너 영역: `AdBanner` 컴포넌트에서 `object-contain` + 고정 높이로 비율 보존
  - 기사 썸네일: 대부분 `object-cover`로 카드 영역을 꽉 채우되, 카드 자체의 고정 높이로 과도한 늘어짐 방지

### 4. 사이드바 & 배너 광고 관리

- **사이드바 구성**
  - 상단 스퀘어 배너
  - 세로형 긴 배너
  - 오른쪽 인기/실시간 추천 슬라이더
- **배너 설정 (관리자 패널)**
  - 로컬스토리지 기반 브랜드 설정(`brandSettings`)
  - 항목:
    - 상단/세로/하단 띠 배너 문구
    - 각 배너 이미지 URL
    - 각 배너 ON/OFF 토글
  - 배너 이미지 업로드:
    - 파일 업로드 시 `/api/admin/brand-assets`로 전송
    - 서버에서 MinIO에 저장 후 공개 URL 반환
    - 파일 이름에 `sidebar-top`, `sidebar-long`, `bottom-banner` 등 배너 위치를 반영하여 추후 관리 용이
  - 관리자 패널 하단에 **현재 사용 중인 배너 이미지 리스트** 표시
    - 썸네일 + URL + 제거 버튼

### 5. 표시/타이포그래피 설정

- 관리자 패널의 **기사 표시 설정**에서 다음을 변경 가능:
  - 본문 글꼴 (Noto Sans, 나눔명조 등)
  - 기본 글자 크기
  - 본문 줄간격
  - 본문 내 이미지 최대 너비
- 이 설정은 `displaySettings`로 로컬에 저장되며, `ArticleDetail` 페이지에서 인라인 스타일로 적용됩니다.

### 6. 추천 슬라이더 (오른쪽 박스)

- 인기/실시간 급상승 탭으로 나뉘어 있으며, 모두 `/api/articles` 응답에서 이미지가 있는 기사만 사용
- 상단:
  - 현재 선택된 기사 한 건을 좌측 썸네일 + 우측 여러 줄 제목 형태로 크게 노출
  - 전체 중 몇 번째인지 `1 / N` 표시
- 하단:
  - 최대 3개의 추천 항목을 세로로 배치
  - 각 항목 클릭 시 상단 카드/상세 페이지가 해당 기사로 전환

### 7. 관리자 패널 (Agent/스케줄/로그/배너)

- 기존 프로젝트의 에이전트 설정/스케줄/로그 관리 UI를 그대로 유지하면서,
  - 기사 표시 설정
  - 브랜드/배너 설정
  를 추가로 제공
- 백엔드 `AdminController`를 통해:
  - 허용 발신자/수정 키워드 관리
  - 스케줄 설정/로그 조회 및 초기화
  - MinIO 기반 브랜드 자산(배너/로고 이미지) 업로드/삭제

## 개발/실행 방법 (요약)

> 실제 포트/DB 설정, MinIO 주소 등은 `.env` 및 `src/newspaper-frontend/.env.*` 파일을 참고하세요.

1. **백엔드**
   - JDK 및 Gradle 환경에서 `src/api-server`를 기본 Spring Boot 방식으로 실행
   - MySQL 및 MinIO가 로컬에서 실행 중이어야 합니다.
2. **프론트엔드**
   - `cd src/newspaper-frontend`
   - `npm install`
   - `npm run dev` 로 개발 서버 실행
3. 브라우저에서 프론트엔드 주소(예: `http://localhost:5173`)로 접속
